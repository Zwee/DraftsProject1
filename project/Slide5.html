Images and Whitespace

|- About Whitespace -|

Whitespace is the parts of the code where there is no code.

Whitespace is all the blank screen around the words. With other applications, like typing up a term paper in MS Word, whitespace can make a big difference. It can break up paragraphs, force text to display on an additional page, or make the text uncomfortable to read. But in HTML, whitespace doesn't matter.(*)

It doesn't matter how many times you hit your 'space' or 'enter' keys: unless you tell the computer where the breaks are, it will show everything in one looooooooong line.
'Break' or <br> is a void tag (and an inline tag) that provides line breaks. It just says 'the line ends here'. (Or, to quote Capt. Jean-Luc Picard: "The line must be drawn here! This far, no further!")

(*) Unless we're talking about class selectors, but more on that later...

...and what on earth is alternate text?

Alt, for 'alternate text', is often used in img (image) tags to provide an alternate output to the image if, for whatever reason, the picture did not download or wasn't available.

Using alternate text is a good habit to get into:

  • It lets users know that there was supposed to be something there, rather than just getting a blank spot on the page (or just a blank page, which is not a very precise error message to de-bug from).
  • The alternate text can tell visually impaired people what's supposed to be there.
  • (Again, note that there is no closing tag for this. <img> is a void tag.)
